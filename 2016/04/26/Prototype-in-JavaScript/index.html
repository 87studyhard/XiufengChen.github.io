Prototype in JavaScript

Apr 26, 2016

原型

原型对象

我们穿件的每个函数都有一个prototype（原型）属性，这个属性是一个指针，指向一个对象，而这个对象的用途是包含可以有特定类型的所有实例共享的属性和方法。意思是，不管创建实例1还是实例2，他们访问的都是同一组属性和方法。

无论什么时候，只要创建了一个新函数，就会根据一组特定的规则为该函数创建一个prototype属性，这个属性指向函数的原型对象。而原型对象包含一个constructor属性，指向原函数。
当调用构造函数创建一个新实例后，该实例的内部将包含一个指针[[Prototype]]（内部属性），指向构造函数的原型对象。


因此，每当代码读取某个对象的某个属性时，都会执行一个搜索，目标是具有这个名字的属性。首先从对象实例开始，如果找到就返回，如果没有找到，则继续搜索指针指向的原型对象。注意，如果实例中和原型中都有某个名字对应的属性，那么根据搜索规则，实例中的属性会屏蔽原型中的属性。
每创建一个函数，就会同时创建它的prototype对象，这个对象自动获得constructor属性，指向创建它的构造函数。事实上，每个对象都有一个constructor属性，指向创建它的构造函数，一般字面量对象创建它的函数当然就是function Object(){}。但是如果将一个新对象赋给函数的prototype对象，即重写prototype对象，那么此处新prototype的constructor属性指向的是Object或者其他构造函数，而非原函数。注意：调用构造函数创建新实例时会为实例添加一个指向原型的[[Prototype]]指针，而重写原型对象就等于切断了构造函数和原型对象之间的联系。如果创建新实例在重写原型对象之前，新原型对象的属性该实例无法使用。如果新实例在重写原型对象之后，新原型对象的属性该实例照常使用。
原生引用类型也是采用原型模式定义方法的，如果想要扩展String类型的方法，直接定义

function XXX(){} //Nothing in constructor funciton
XXX.prototype.yyy = 
XXX.prototype.yyy =

清楚原型模式的优缺点之后，创建自定义类型一般情况下组合使用构造函数模式和原型模式。构造函数模式用于定义实例属性，而原型模式用于定义方法和共享属性。

####原型链
原型链是实现集成的主要方法，利用原型让一个引用类型继承另一个引用类型的属性和方法，实现的本质是重写原型对象，代之以一个新类型的实例。
